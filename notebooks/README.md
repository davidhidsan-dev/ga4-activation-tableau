# notebooks/

## ES — Por qué esta carpeta está vacía
En este proyecto no se usaron notebooks como entorno principal de análisis.

**Motivo:** el flujo se diseñó como un caso “empresa real” centrado en:
- **BigQuery SQL** para extracción, agregaciones, segmentación y **BigQuery ML (BQML)**.
- **Tableau Public** para visualización y storytelling.

La “fuente de verdad” del proyecto está en:
- `/sql` (queries)
- `/docs` y `/reports` (metodología, definiciones e interpretación)

Los notebooks se reservarían para:
- prototipos rápidos,
- análisis ad-hoc,
- o modelos más avanzados fuera de BQML.

> Si en el futuro se amplía el proyecto, aquí se añadirían notebooks para EDA adicional o benchmarking de modelos en Python.
